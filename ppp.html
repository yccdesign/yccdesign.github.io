The Project

Developing a customized experience for a govermental project


For a semester-based extracurricular UX consulting initiative, I worked in a team of two data scientists and three UX designers to provide UX service for Dynamo Metrics, a data visualization company.

The company's new project is a tailor-made data visualization application for the code enforcement team at the City of Jackson. With little UX background and resource, the company is in need of professioal insights to deliver optimal solution. Our deliverables were to conduct user research and provide a mid-fi prototype



At the end of the semester project, I am offered a semester-long internship to continue the interation with another UX designer in the team.

The Redesign

Presenting only my contribution


After a series of user research conducted in group, I am solely responsible for redesigning the neighborhood data visualization feature for the project, while the other two designers each took charge of the design of one other features of the application. The new design of the feature has 5 key revisions.


Neighborhood Data On Hover

Include instant summary of neighborhood data when users hover over the neighborhood on map. An Addition of changeability index is included in the statistics for instant target identification.

New Layer Adding Process

Reduced visual distraction on UI, presenting only necessary information.

Draggable Layer Design

Added drag-and-drop features for each layers for intuitive navigation and data interpretation.

Legend Panel on Map

Added legend panel with collapsible details on the map for more efficient interpretation.

Redesigned Historic Trend Playback

Rearranged configurations for more streamlined experience.

User Research

Before designing the prototype, I worked with other team members on a series of user research.



1. Competitive Analysis


The team first started out with competitive analysis, where I have analysized two other urban data visualization products on the market. From the analysis, we especially found our client's current application would improve by adding predictive and summative data for users to quickly learn the status of an area.




2. User Interviews


Four interviews were conducted at the City of Jackson with potential users in the government. Prior to the interview, the whole team worked together on a semi-structured interview protocol with an aim to elicit specific details regarding the interviewee's background, job responsibilities, daily workflow, challenges, and expectations.



After the interview, the UX team interpreted the data and organized the notes into a structured affinity wall, including:


  • Daily tasks
  • Duties
  • Issues regarding workflow
  • Issues regarding software tools
  • Data types in need


3. Personas


After the interviews, each UX designer developed a persona, each representing a type of the end users: code enforcement director, code enforcement officer, and administrator.



4. Workflow


Prior to the design phase, the UX team developed a flowchart that details the daily routine of the end users. With the help of the flow chart, we were able to efficiently design an interface collaborately.
